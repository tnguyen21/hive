# Plan: Add Mutation Testing to Hive

## Context

Hive has a solid test suite (19 files, pytest + xdist parallel, async support) but no way to measure test *quality*. Mutation testing systematically introduces small code changes (mutants) and checks whether tests catch them. Surviving mutants reveal weak assertions, missing edge cases, and undertested branches. The recent refactoring pass has stabilized the codebase — good time to add this before new feature work.

## Tool: mutmut

**Why mutmut over alternatives:**
- Native `pyproject.toml` config (`[tool.mutmut]`)
- Built-in `--parallel` for running multiple mutations concurrently
- Simplest setup of the three options (cosmic-ray needs Celery, mutatest is immature)
- Works fine with async Python — mutations are syntactic, not runtime

**Key constraint:** xdist (`-n auto`) must be disabled for mutation runs. xdist's per-run process spawn overhead (~0.5s) dominates when each run is short due to `-x` fail-fast. mutmut's `--parallel` handles concurrency at the mutation level instead.

## Changes

### 1. Add mutmut dependency and config

**`pyproject.toml`** — add `"mutmut>=3.0"` to both dep groups, add config section:

```toml
[tool.mutmut]
paths_to_mutate = [
    "src/hive/status.py",
    "src/hive/config.py",
    "src/hive/utils.py",
    "src/hive/git.py",
    "src/hive/db/core.py",
    "src/hive/db/issues.py",
    "src/hive/db/metrics.py",
    "src/hive/db/notes.py",
    "src/hive/orchestrator/completion.py",
]
tests_dir = "tests/"
runner = "python -m pytest -x -n 0 --timeout=10 -q -m 'not integration'"
```

Runner flags:
- `-x`: fail-fast (biggest perf win — stop on first killed test)
- `-n 0`: disable xdist per-mutation
- `--timeout=10`: kill infinite-loop mutants
- `-m 'not integration'`: skip slow integration tests

### 2. Add `.mutmut-cache/` to `.gitignore`

mutmut stores results in a local SQLite cache.

### 3. Create `scripts/mutate.sh`

Convenience wrapper (the `scripts/` directory doesn't exist yet — create it):

```bash
#!/usr/bin/env bash
# Usage:
#   ./scripts/mutate.sh              # Full run
#   ./scripts/mutate.sh status.py    # Single module (matches path suffix)
#   ./scripts/mutate.sh --results    # Show surviving mutants
```

- Accepts a module shorthand, resolves to full path under `src/hive/`
- Runs `uv run mutmut run --parallel`
- Prints surviving mutant count at the end

### 4. Fill critical test gaps (pre-mutation)

Without these, the first mutation run would be overwhelmed by survivors from untested code.

**`tests/test_status.py`** (new) — tests for:

- `session_status_payload`: correct dict structure
- `UNBLOCKING_ISSUE_STATUSES` / `CLOSED_ISSUE_STATUSES` membership (regression guards)

**`tests/test_config.py`** (additions) — `_coerce()` edge cases:
- `_coerce(None, int)` -> `None`
- `_coerce("false", bool)` -> `False`

- `_coerce("yes", bool)` -> `True`
- `_coerce(True, bool)` -> `True` (isinstance path)
