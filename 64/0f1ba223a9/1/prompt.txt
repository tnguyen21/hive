# Plan: Retry Counter Reset via Watermark Event

## Context

When an issue repeatedly fails/escalates, the retry counter (computed by counting `retry`, `agent_switch`, and `incomplete` events) prevents it from being retried further. After fixing the underlying cause, there's no CLI way to reset these counters — users must manually delete rows in SQLite. We'll add a "watermark" event (`retry_reset`) that resets the counters without destroying the audit trail.

## Changes

### 1. Add `--reset` flag to `hive retry` CLI command

**`src/hive/cli/parser.py`** (~line 147):
- Add `--reset` flag to the existing `retry` parser

**`src/hive/cli/parser.py`** (~line 360, dispatch):
- Pass `args.reset` through to `cli.retry()`

**`src/hive/cli/core.py`** (~line 449, `retry()` method):
- Accept `reset: bool = False` parameter
- When `reset=True`, log a `retry_reset` event (instead of / in addition to the `manual_retry` event)
- Include notes in the event detail

### 2. Add `count_events_by_type_since_reset()` to DB layer

**`src/hive/db/metrics.py`** (after `count_events_by_type`):
- New method `count_events_by_type_since_reset(issue_id, event_type)` that counts events only after the most recent `retry_reset` event:
```sql
SELECT COUNT(*) FROM events
WHERE issue_id = ? AND event_type = ?
  AND created_at > COALESCE(
    (SELECT MAX(created_at) FROM events
     WHERE issue_id = ? AND event_type = 'retry_reset'),
    '1970-01-01'
  )
```

### 3. Replace `count_events_by_type` calls with reset-aware version

All call sites that feed into escalation decisions need the new method:

**`src/hive/orchestrator/completion.py`** (lines 326, 330, 375, 390, 404-405):
- `_choose_escalation()` — 2 calls (retry + agent_switch)
- `_handle_agent_failure()` — 4 calls (retry count for detail, agent_switch count for detail, and 2 for the final escalation detail)

**`src/hive/orchestrator/core.py`** (lines 256-257):
- `_reconcile_stale_agents()` — 2 calls (retry + agent_switch budget check)

### 4. Filter `build_retry_context()` to respect the watermark

**`src/hive/prompts.py`** (~line 64):
- Query for the most recent `retry_reset` event's `created_at`
- Filter `incomplete`, `merge_rejected`, and `stalled` events to only include those after the reset timestamp
- This way agents don't see stale failure context from before the reset

### 5. `agent_runs` view — leave as-is

The `agent_runs` view (line 216 of `core.py`) uses a raw subquery for `retry_count`. This is a **historical metrics view** — it should reflect total lifetime retries, not just post-reset ones. No change needed.
