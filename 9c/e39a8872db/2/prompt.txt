# Plan: Global Multi-Project Status View

## Context

`hive status` currently only works inside a project (git repo). Since start/stop
now work globally via `initialize_global()`, we should extend `status` to show a
multi-project dashboard when run outside a project. The daemon is global and
serves all registered projects — users need visibility into all of them from
anywhere.

## Approach

The `status` command in `typer_app.py` tries `resolve_project()`. If it succeeds,
use existing single-project path. If it fails (no git repo), fall back to
`initialize_global()` and build a global view from all registered projects in the DB.

## Changes

### 1. New module: `src/hive/cli/global_status.py`

Single function `get_global_status(db: Database) -> dict` that:

- Gets daemon status via `HiveDaemon(db_path=db.db_path).status()`
- Calls `db.list_projects()` to get all registered projects
- For each project:
  - Checks `Path(project["path"]).exists()` — if missing, sets `path_missing: True` and skips queries
  - Queries issue counts: `SELECT status, COUNT(*) FROM issues WHERE project = ? GROUP BY status`
  - Queries active agents: `db.get_active_agents(project=name)`, enriched with issue titles via `db.get_issue()`
  - Queries refinery: `SELECT mq.issue_id, i.title FROM merge_queue mq LEFT JOIN issues i ON mq.issue_id = i.id WHERE mq.status = 'running' AND mq.project = ?`
  - Queries merge stats: new `db.get_merge_queue_stats(project=name)` (see below)
  - Queries escalated issues: `SELECT id, title FROM issues WHERE project = ? AND status = 'escalated'`
  - Git worktree check: only if project has queued/running merges, otherwise `status: "unchecked"`
- Computes aggregate totals (open, in_progress, done, escalated, workers) across all projects

Returns:
```python
{
    "daemon": {"running": bool, "pid": int|None, "log_file": str},
    "totals": {"open": N, "in_progress": N, "done": N, "escalated": N, "workers": N},
    "projects": [
        {
            "name": str, "path": str, "path_missing": bool,
            "issues": {status: count, ...}, "total_issues": N,
            "active_agents": N,
            "workers": [{"name": str, "issue_id": str, "issue_title": str}],
            "refinery": {"active": bool, "issue_id": str|None, "issue_title": str|None},
            "merge_queue": {"queued": N, "running": N, "merged": N, "failed": N},
            "merge_blockers": [...],
            "main_worktree": {"dirty": bool, "status": str, ...},
            "attention_issues": [{"id": str, "title": str}],
        },
        ...
    ],
}
```

### 2. Modify `db.get_merge_queue_stats()` in `src/hive/db/core.py`

Add optional `project` parameter to existing method:

```python
def get_merge_queue_stats(self, project: str | None = None) -> dict[str, int]:
    if project:
        cursor = self.conn.execute(
            "SELECT status, COUNT(*) as count FROM merge_queue WHERE project = ? GROUP BY status",
            (project,),
        )
    else:
        cursor = self.conn.execute(
            "SELECT status, COUNT(*) as count FROM merge_queue GROUP BY status"
        )
    stats = {"queued": 0, "running": 0, "merged": 0, "failed": 0}
    for row in cursor.fetchall():
        stats[row["status"]] = row["count"]
    return stats
```

### 4. Rewrite `status` command in `src/hive/cli/typer_app.py`

Same pattern as `start`/`stop`: try project detection, fall back to global.

```python
@app.command()
def status(ctx: typer.Context) -> None:
    """Show orchestrator status."""
    try:
        resolve_project(ctx.obj.project)
        # Project detected — use existing single-project path
        _run(ctx, "status")
        return
    except (SystemExit, Exception):
        pass

    # No project context — show global multi-project view
    db = initialize_global(db_override=ctx.obj.db_override)
    try:
        from .global_status import get_global_status
        result = get_global_status(db)
        if ctx.obj.json_mode:
            ctx.obj.console.print_json(json.dumps(result))
        else:
            from .rich_views import render_global_status
            ctx.obj.console.print(render_global_status(result))
    finally:
        db.close()
```
